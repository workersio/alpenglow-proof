    Pulling back rings along injective maps, and pushing them forward along surjective maps #

    Implementation note #

    The nsmul and zsmul assumptions on any transfer definition for an algebraic structure involving both addition and multiplication (e.g. AddMonoidWithOne) is ∀ n x, f (n • x) = n • f x, which is what we would expect. However, we cannot do the same for transfer definitions built using to_additive (e.g. AddMonoid) as we want the multiplicative versions to be ∀ x n, f (x ^ n) = f x ^ n. As a result, we must use Function.swap when using additivised transfer definitions in non-additivised ones.
