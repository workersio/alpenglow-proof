    Facts about (co)limits of functors into concrete categories #

    The forgetful functor on Type u is the identity; copy the instances on ùü≠ (Type u) over to forget (Type u).

    We currently have two instances for HasForget (Type u):

    • A global HasForget instance where forget (Type u) reduces to ùü≠ Type
    • A locally enabled ConcreteCategory where forget (Type u) is only reducible-with-instances equal to ùü≠ Type.

    Since instance synthesis only looks through reducible definitions, we need to help it out by copying over the instances that wouldn't be found otherwise.
